### 2.8 Site Registration Process

---

#### Purpose

Site registration does three things: it assigns a site code and facilitator code that are used across all data collection; it adds the site to the public project map on the WordPress site; and it triggers delivery of the QR code card set to the facilitator.

---

#### When to Register

After the facilitator has:
- Confirmed their exhibition venue in writing
- Recruited at least four confirmed participants in each cohort
- Confirmed their youth co-facilitator

Registration should happen at least two weeks before Week 1 to allow time for QR code generation and delivery.

---

#### How to Register

#### How to Register

A Contact Form 7 registration form on the WordPress site at `[REGISTRATION_FORM_URL]`. Fields as listed below. On submission, the form sends an email to the project coordinator who reviews the registration within 48 hours.

Facilitators do not receive an automated response on submission - the coordinator contacts them directly once the registration is approved and the site has been entered into the project database. 

Fields:

*Facilitator details*
- Name and email address
- Location - town/suburb, state/territory, country
- Organisation name if any - optional
- Brief description of how they heard about the project

*Site details*
- Proposed start date
- Proposed weekly session day
- Exhibition venue type and name
- Youth cohort recruitment pathway
- Older adult cohort recruitment pathway
- Youth co-facilitator name, role, and Working With Children Check or equivalent status

*Confirmation checkboxes*
- I have read the Facilitator Guide
- I have confirmed my exhibition venue in writing
- I have confirmed my youth co-facilitator
- I understand that data submission is required on project completion
- I accept the project's data handling terms as described in the Facilitator Guide

---

#### What Happens After Registration

1. Project coordinator receives registration by email and reviews within 48 hours
2. Coordinator enters the approved registration into the project database
3. Site code, facilitator code, and status update key are assigned automatically
4. Coordinator emails the facilitator with:
   - Site code and facilitator code
   - Status update key and status update link
   - Sixteen QR code cards as a print-ready PDF - one set per cohort, eight weekly cards each
   - Link to download the kit
5. Site appears on the WordPress project map as "registered - not yet started"
6. Facilitator added to the project mailing list for kit updates

---

#### Status Tracking

Each registered site has one of four statuses on the project map:
- Registered - not yet started
- In progress - week number shown
- Completed - data submitted
- Incomplete - project did not finish; brief reason noted

Facilitators update their own site status via a simple form linked from their registration confirmation email.
